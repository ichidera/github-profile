<sub>This animation is generated by a GitHub Action that runs in your profile repo, reads your contribution graph, and auto-commits an SVG. Unlike external APIs, it has zero ongoing dependency — setup takes ~2 min.</sub>

<details>
<summary><strong>✅ Setup Instructions</strong> (required once)</summary>

**1. Create `.github/workflows/snake.yml` in your profile repo:**

```yaml
name: Generate Contribution Snake

on:
  schedule:
    - cron: "0 */6 * * *"  # Every 6 hours
  workflow_dispatch:
  push:
    branches: [main]

jobs:
  generate:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - name: Generate snake animation
        uses: Platane/snk@v3
        with:
          github_user_name: ${{ github.repository_owner }}
          outputs: |
            dist/github-contribution-grid-snake.svg
            dist/github-contribution-grid-snake-dark.svg?palette=github-dark
      
      - name: Push to output branch
        uses: crazy-max/ghaction-github-pages@v3
        with:
          target_branch: output
          build_dir: dist
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**2. Push the workflow to your repo:**
```bash
git add .github/workflows/snake.yml
git commit -m "feat: add contribution snake animation"
git push
```

**3. First run:**
- GitHub Actions will automatically trigger and run the workflow
- After ~2 min, check the `output` branch exists in your repo
- The SVG file will be at: `https://raw.githubusercontent.com/YOUR_USERNAME/YOUR_USERNAME/output/github-contribution-grid-snake.svg`

**4. Troubleshooting:**
- Action shows "failed"? → Check repo permissions: Settings → Actions → General → Workflow permissions must be "Read and write"
- `output` branch missing? → Manually trigger: go to Actions tab → "Generate Contribution Snake" → "Run workflow"
- Still blank? → Update the image src in README to match your actual `output` branch URL

</details>
